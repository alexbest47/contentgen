
Итоговый шаблон = результат генерации в сервисе = используется в Wordpress как шаблон страницы. Эталонный образец кода находится в na-demo.php

Правила работы с шаблоном:
Есть части кода, которые не меняются и подключены штатными средствами WP. В итоговом шаблоне оставлять код подключения из эталона. Для отображения в сервисе брать данные по матрице подключения:
get_header('light') = header-light.txt
get_footer('light') = footer-light.txt
get_template_part('inc/landing-breadcrumbs', null, ['psychology', 'Курсы психологии']) = landing-breadcrumbs.txt
get_template_part( 'inc/components/form/agreed' ) = agreed.txt

Пути до статических файлов (стилей, скриптов, изображений, шрифтов и т.д.) указаны относительно корня домена talentsy.ru Для отображения в сервисе подставлять домен, в итоговом шаблоне оставлять относительный путь. В случае, если добавляют новое изображение из внешнего хранилища - указывать полный путь.

Изменения стилей добавлять inline в конце шаблона до get_footer('light'), использовать неконфликтные классы для элементов.

Эталон содержит переменные $programs, $faq, $teachers, $coursePrices - это acf поля с массивами данных. Сейчас планируется отказаться от получения данных из панели управления, всё хардкодом из редактора. В итоговом шаблоне ожидается логика без acf полей, переменные из начала эталона нужно убрать.

Итоговый шаблон на строке 4 должен содержать уникальное название, заменяется только слово "Demo" на нужное. Если итоговый шаблон для нового ледндига - оно генерируется по названию, если шаблон отправлен как обновление существующего - менять название шаблона нельзя.

Код обработки форм ($isGetCourse и $formParmas) изменять запрещено. Для форм обязательно выводить так, как в эталонном образце.
