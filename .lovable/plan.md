

## Задача

Блоки «Программы» и «Бесплатные курсы» используют неверный маппинг цветов из схемы: ссылки получают `preview_colors[1]` (светлый фон `#F0EDF7`), который почти не виден. Нужно: белый фон блоков, корректные цвета текста и ссылок.

## Анализ цветов схемы «Классика»

```text
preview_colors[0] = #7B2FBE  ← фиолетовый (бренд/акцент)
preview_colors[1] = #F0EDF7  ← светлая лаванда (фон)
preview_colors[2] = #1A1A2E  ← тёмный (текст)
preview_colors[3] = #FFFFFF  ← белый
```

Текущий маппинг в обоих компонентах:
- `headingColor = [0]` → фиолетовый ✓
- `accentColor = [1]` → светлая лаванда ✗ (должен быть [0] для ссылок)

## Изменения

### 1. `PaidProgramsCollectionSettings.tsx`

- Фиксировать маппинг: заголовки `preview_colors[2]` (тёмный), ссылки `preview_colors[0]` (акцент), разделители — светло-серый
- Описание и теги оставить `#444444` / `#888888` — стабильные нейтральные цвета
- Обернуть весь блок в `<table>` с белым фоном (`background:#FFFFFF`)

### 2. `FreeCoursesGridSettings.tsx`

- Аналогичный маппинг: заголовки `[2]`, ссылки `[0]`, фон плейсхолдера `[1]`
- Обернуть в таблицу с белым фоном

### 3. `BlockCanvas.tsx`

- Для блоков с `generated_html` типа `paid_programs_collection` и `free_courses_grid` — рендерить контейнер с `background: white`
