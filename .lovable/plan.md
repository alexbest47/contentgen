

## Визуальное отображение структуры шаблона в пустом письме

Сейчас wizard создаёт блоки из шаблона (с типами `hook_monologue`, `offer_cta` и т.д.), но канвас не знает их названий — они отображаются как пустые безымянные блоки.

Нужно: показать красивые read-only плашки с названиями блоков из шаблона, без возможности редактирования/перемещения/удаления.

---

### 1. Сохранять `label` из шаблона при создании блоков (`CreateLetterWizard.tsx`)

При `insert` в `email_letter_blocks` добавить `label` из шаблона в `config`:

```js
config: { mode: b.mode || "text_only", label: b.label }
```

Это единственное изменение в wizard — label будет храниться прямо в config блока.

### 2. Канвас: показать шаблонные блоки как read-only плашки (`BlockCanvas.tsx`)

В рендеринге блоков, когда блок пустой (нет `generated_html`, не user-block), проверяем `block.config.label`:

- Если есть `config.label` — показываем read-only плашку с иконкой замка, названием из `config.label` и подписью «будет сгенерировано».
- Скрываем hover-контролы (перемещение, удаление, настройки) для таких блоков — структура фиксирована шаблоном.

### 3. Что НЕ меняется

- Не нужно регистрировать новые типы в `BlockLibrary`.
- `isTemplateLocked` и `isGeneratedBlock` не трогаем.
- Правая панель (выбор кейса/возражений) — без изменений.

Итого: правки в 2 файлах, ~15 строк.
