

## Добавление произвольного типа баннера с поддержкой изображений

### Что делаем

1. Добавить тип баннера `custom` ("Произвольный") без фиксированных размеров.
2. При выборе этого типа скрывать поля: категория, программа, цветовая гамма.
3. Добавить возможность прикрепить reference-изображение к промпту (drag-and-drop или выбор файла).
4. Передавать изображение в edge-функцию как base64 и отправлять в OpenRouter вместе с промптом.

### Файлы и изменения

**1. `src/lib/bannerConstants.ts`**
- Добавить `{ key: "custom", label: "Произвольный", width: 0, height: 0 }` в `BANNER_TYPES`.
- Добавить пустой шаблон промпта для `custom`.

**2. `src/components/banners/AddBannerDialog.tsx`**
- При `bannerType === "custom"` скрывать блоки: категория, программа/оффер, цветовая гамма.
- Добавить поле для загрузки reference-изображения (отдельное от основного upload) — показывать только во вкладке «Сгенерировать» при любом типе баннера.
- Конвертировать загруженное reference-изображение в base64 и передавать в payload как `reference_image`.
- При `custom` — не требовать цветовую гамму для кнопки «Сгенерировать».

**3. `supabase/functions/generate-banner-image/index.ts`**
- Добавить `custom` в `BANNER_SIZES` (fallback 1024×1024 или без ограничений).
- Если в payload есть `reference_image` (base64 строка), формировать `messages` как мультимодальный запрос:
  ```ts
  messages: [{
    role: "user",
    content: [
      { type: "image_url", image_url: { url: referenceImage } },
      { type: "text", text: fullPrompt }
    ]
  }]
  ```
- При `banner_type === "custom"` — не добавлять PREAMBLE и image_style (чистый промпт).
- При `custom` — не пытаться загружать цветовую гамму.

### Логика UI при `custom`

```text
Тип = "Произвольный" выбран:
  ├── Название *          — показывается
  ├── Категория           — скрыта
  ├── Программа/Оффер     — скрыта  
  ├── Цветовая гамма      — скрыта
  ├── Reference-изображение — показывается (опционально)
  ├── Промпт *            — показывается (без шаблона, свободный текст)
  └── Заметка             — показывается
```

### Ограничения

- Максимальный размер reference-изображения: 5 MB (конвертируется в base64 ~6.7 MB).
- Для custom-баннеров в БД `category` ставится `"paid_program"` по умолчанию (поле обязательное в таблице), но это чисто техническая деталь.
