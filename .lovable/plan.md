

## Ускорить загрузку страницы «Очередь задач»

### Проблема

1. **Клиентская пагинация**: загружаются все 100 записей, затем нарезаются на страницы в браузере
2. **Realtime без дебаунса**: каждое изменение в `task_queue` (а при генерации цепочки из 16 писем их много) вызывает полный `fetchTasks()` — каскад тяжёлых запросов
3. **`setLoading(true)` при каждом обновлении**: таблица мигает при каждом realtime-событии

### Решение

**Серверная пагинация** + **дебаунс realtime** + **убрать мигание при фоновом обновлении**

### Изменения в `src/pages/TaskQueue.tsx`

1. Использовать `.range((page-1)*PAGE_SIZE, page*PAGE_SIZE - 1)` вместо `.limit(100)` + клиентского `slice`
2. Добавить `{ count: "exact", head: false }` в `select` для получения общего количества записей
3. Обернуть realtime-callback в `setTimeout`/debounce (300мс) — чтобы пачка событий вызывала один запрос
4. Показывать `loading` только при первой загрузке, а при realtime-обновлениях обновлять данные без мигания (silent refresh)
5. Добавить фильтр по `lane` (табы уже есть по статусу)

| Файл | Изменение |
|------|-----------|
| `src/pages/TaskQueue.tsx` | Серверная пагинация, дебаунс realtime, silent refresh |
