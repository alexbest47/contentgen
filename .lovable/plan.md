

## Упростить переменные офферов: промокоды, освободилось место, старт нового потока

### Анализ

Все три типа офферов уже используют стандартные offer-переменные:
- `{{offer_title}}` — название (заполняется для всех трёх типов)
- `{{offer_value}}` — короткое описание (заполняется для промокодов через `description`)
- `{{offer_description}}` — полное описание из Google Docs (не заполняется ни для одного из этих типов — у них нет `doc_url` в форме)

Таким образом, специализированные переменные должны содержать **только уникальные поля**, которых нет в стандартных offer-переменных.

### Изменения

**`src/pages/PromptVariables.tsx`**

1. **`{{promo_codes_data}}`**: убрать `program` и `description`, оставить только `code` и `expires_at`. Обновить описание в каталоге.

2. **`{{spot_available_data}}`**: удалить полностью — у этого типа нет уникальных полей (название уже в `{{offer_title}}`). Убрать из `categories`, удалить `SpotAvailableCard`, убрать из рендера.

3. **`{{new_stream_data}}`**: убрать `program` и `title`, оставить только `stream_start_date`. Обновить описание.
