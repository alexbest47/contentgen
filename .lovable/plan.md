### 1. Миграция БД
Добавить колонку `name` (text, nullable) в таблицу `case_jobs`:
```sql
ALTER TABLE public.case_jobs ADD COLUMN name text;
```

### 2. Изменения в `src/pages/CaseManagement.tsx`

**Форма создания задачи:**
- Добавить поле ввода `jobName` (state) над полем URL
- Передавать `name` при создании — после вызова `scan-case-folder` обновлять запись `case_jobs` с `name`

**Список задач — отображение и редактирование:**
- В каждой карточке задачи: показывать `job.name` как заголовок (если есть), иначе показывать `folder_url`
- Добавить инлайн-редактирование названия: клик по названию или иконка «карандаш» → поле ввода → сохранение через `supabase.from("case_jobs").update({ name }).eq("id", job.id)`
- Редактирование доступно для всех задач (не только завершённых)

### Детали реализации
- Новый state: `jobName` для формы, `editingJobId` + `editingName` для инлайн-редактирования
- `updateNameMutation` — мутация для обновления имени
- В `startMutation.onSuccess` — если `jobName` был заполнен, сразу обновить запись
- Альтернативно: передать `name` в edge function и сохранить при создании job (чище, но требует изменения edge function)

Предпочтительный подход: обновлять `name` на клиенте после создания job (минимум изменений в edge function).
