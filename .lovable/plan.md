

## Показ уже сгенерированных результатов во время генерации и после остановки

Сейчас блоки результатов (quiz JSON, thank you JSON, card prompt) показываются только при `status === "ready"` (строка 574: `isReady`). Нужно показывать их также при `generating_images`, `card_prompt_generated`, `generating_card_prompt`, `quiz_generated` и `error` — если соответствующие данные уже есть в БД.

### Изменения в `src/pages/DiagnosticDetail.tsx`

1. **Убрать привязку результатов к `isReady`**. Вместо `{isReady && (...)}` на строке 574 показывать каждый блок результатов независимо, по наличию данных: `{quizJson && (...)}`, `{thankYouJson && (...)}`, `{cardPrompt && (...)}`.

2. **Polling должен подгружать все поля**. Сейчас polling (строка 142) загружает `status, generation_progress, quiz_json`. Добавить `thank_you_json, card_prompt` чтобы UI обновлялся при появлении новых данных. После получения — обновлять кэш React Query.

3. **Результаты показываются ниже прогресса и ошибки** — порядок секций остаётся тем же, просто снимается условие `isReady`.

4. Кнопку «Создать ещё» показывать только при `isReady`.
