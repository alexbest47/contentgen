

## Задача

Вернуть кнопки «Сгенерировать» прямо в область каждого image-плейсхолдера внутри письма, вместо отдельной панели внизу.

## Проблема

`contentEditable` div не позволяет вставлять React-компоненты внутрь. Сейчас кнопки вынесены в отдельную панель под письмом.

## Решение

Использовать **overlay-подход**: после рендера HTML через `useRef` + `useEffect` находить все `[data-placeholder-id]` элементы в DOM и позиционировать React-кнопки поверх них абсолютно.

### `BlockCanvas.tsx`

1. Добавить `useRef` на contentEditable div и `useEffect`, который после каждого рендера:
   - Ищет все `div[data-placeholder-id]` внутри контейнера
   - Для каждого вычисляет `offsetTop`/`offsetLeft`/`width`/`height` относительно контейнера
   - Сохраняет координаты в state: `Array<{id, top, left, width, height}>`

2. Рендерить кнопки генерации как абсолютно позиционированные элементы поверх contentEditable div (обернуть в `relative` контейнер). Каждая кнопка центрирована внутри соответствующего плейсхолдера. Кнопки имеют `pointer-events: auto`, а overlay-контейнер — `pointer-events: none`, чтобы не мешать редактированию текста.

3. Удалить панель «Изображения для генерации» снизу (строки 122-148).

### Результат

```text
┌──────────────────────────────┐
│  [placeholder area]          │
│    header_banner — 600x200   │
│    [ 🖼 Сгенерировать ]      │  ← кнопка поверх заглушки
│                              │
├──────────────────────────────┤
│  Текст письма (editable)     │
├──────────────────────────────┤
│  [placeholder area]          │
│    schema_image — 600x300    │
│    [ 🖼 Сгенерировать ]      │
└──────────────────────────────┘
```

Один файл: `BlockCanvas.tsx`.
