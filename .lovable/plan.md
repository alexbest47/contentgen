

## Fix: Sidebar link «Работа с возражениями» ведёт на несуществующий роут

### Проблема
Сайдбар ссылается на `/objections`, но маршрут определён как `/programs/:programId/objections`. Без `programId` — 404.

### Решение

Два варианта:

**Вариант A (рекомендуемый):** Убрать пункт из общего сайдбара. Доступ к возражениям — только через страницу конкретной программы (`ProgramDetail`), где уже есть кнопка «Работа с возражениями». Это логично, т.к. возражения привязаны к программе.

**Вариант B:** Создать страницу-хаб `/objections`, которая показывает список программ и позволяет выбрать, к какой перейти. Избыточно.

### Изменения (Вариант A)
- **`src/components/AppSidebar.tsx`**: удалить строку `{ title: "Работа с возражениями", url: "/objections", icon: ShieldQuestion }` из `contentPrepNav`
- **`src/App.tsx`**: удалить маршрут `/objections` (если есть отдельный), оставить только `/programs/:programId/objections`
