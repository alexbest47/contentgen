

## Перенос кнопки «Создать диагностику» в сайдбар администратора

### Что меняется

1. **`src/pages/OfferTypeDetail.tsx`** — убрать блок с кнопкой «Создать оффер» и диалогом создания (строки 260-280, а также связанные состояния и мутацию создания). Оставить только список офферов, редактирование и архивацию. Изменить пустое состояние на «Нет офферов» без призыва создать.

2. **`src/components/AppSidebar.tsx`** — добавить в `adminNav` пункт «Создать диагностику» с иконкой `Stethoscope` и url `/create-diagnostic`.

3. **Новая страница `src/pages/CreateDiagnostic.tsx`** — содержит:
   - Выпадающий список (Select) для выбора программы из `paid_programs`
   - Форму создания оффера (переиспользуя ту же логику: название, описание, ссылка на Google Doc, теги)
   - При создании `offer_type` автоматически выставляется в `"diagnostic"`
   - После успешного создания — редирект на страницу оффера

4. **`src/App.tsx`** — добавить маршрут `/create-diagnostic` как `adminOnly`.

### Технические детали

- Форма `OfferForm` из `OfferTypeDetail.tsx` будет извлечена в общий компонент или скопирована в новую страницу (проще скопировать, т.к. она небольшая)
- Select программы загружает `paid_programs` через `useQuery`
- Мутация создания вставляет в `offers` с `offer_type: "diagnostic"` и выбранным `program_id`
- После создания `navigate` ведёт на `/programs/:programId/offers/diagnostic/:offerId`
