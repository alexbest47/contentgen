

## Задача

4 проблемы:
1. Первый баннер (header_banner) не генерируется — ошибка при вызове API
2. Пользовательские блоки не выровнены с генерируемым контентом (padding расхождение)
3. Лид-магнит ожидается 600×400, но в placeholder stubs показывается старый размер
4. Нужны кнопки «Перегенерировать» и «Загрузить своё» справа от каждого сгенерированного изображения

## Анализ

**Баннер**: `image_placeholder_1` имеет пустой `image_url`. Промпт содержит текст про психологию/созависимость — возможно safety filter OpenRouter отклоняет. Нужно добавить retry с упрощенным промптом и логирование ошибки на фронт.

**Выравнивание**: Генерируемый HTML рендерится в `contentEditable` div с `p-4` (16px padding). Пользовательские блоки (текст, CTA, изображения) рендерятся в отдельных `div.p-4` карточках ниже — итого у них двойной отступ (16px от карточки). Нужно убрать padding у пользовательских блоков в full-letter mode или сделать его идентичным.

**Размер лид-магнита**: Это вопрос промпта, не кода. В placeholder stub показывается `ph.size`, который приходит из AI. Код корректен — проблема в промпте, который задает `600x300`. Но можно на уровне UI ничего не менять — размер определяется промптом.

**Кнопки рядом с изображениями**: Сейчас кнопка «Сгенерировать» позиционируется поверх placeholder. Нужно: для уже заполненных placeholders (с `image_url`) показывать 2 кнопки справа от изображения (за пределами 600px контейнера). Также нужен upload handler.
