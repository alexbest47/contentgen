

## Проблема

Сейчас клик по диагностике в списке офферов программы ведёт на `/diagnostics/${d.id}` (админ-страница генерации теста). Нужно вместо этого вести на страницу оффера (`OfferDetail`), где можно создавать проекты и генерировать лид-магниты — точно так же, как для мини-курсов.

## Решение

Одно простое изменение в `src/pages/OfferTypeDetail.tsx`:

**Заменить навигацию** при клике по диагностике:

```typescript
// Было:
onClick={() => navigate(`/diagnostics/${d.id}`)}

// Станет:
onClick={() => navigate(`/programs/${programId}/offers/diagnostic/${d.offer_id}`)}
```

Это направит пользователя на существующую страницу `OfferDetail`, которая:
- Показывает список проектов для оффера
- Имеет кнопку «Сгенерировать лид-магниты»
- Позволяет перейти в проект для выбора лид-магнита и генерации контента

Страница `OfferDetail` уже полностью поддерживает этот флоу — она загружает оффер по `offerId` и показывает проекты, привязанные к нему.

**Примечание**: нужно добавить проверку на наличие `offer_id` у диагностики (он может быть `null` для старых записей) и показывать предупреждение в таком случае.
