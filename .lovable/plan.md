

## Задача

Выровнять текст пользовательских блоков (платные программы, бесплатные мини-курсы) с текстом сгенерированного через промпт контента.

## Анализ

- **ContentEditable** (AI-контент): обёрнут в `div.p-4` → 16px padding со всех сторон. Внутри — HTML-таблицы шириной 600px с собственными inline-стилями (padding в ячейках).
- **Пользовательские блоки**: обёрнуты в `div.px-4.py-1` → 16px горизонтальный padding. Внутри — тоже HTML-таблицы (из `PaidProgramsCollectionSettings`, `FreeCoursesGridSettings`) с собственными inline-стилями.

Проблема: пользовательские блоки получают **дополнительные 16px** горизонтального padding от wrapper'а `px-4`, которого нет у AI-контента (у того padding применяется к contentEditable, но таблицы внутри занимают всю ширину).

## Изменение

### `src/components/email-builder/BlockCanvas.tsx` (строка 391)

Убрать `px-4 py-1` для full letter mode, заменить на `py-1` (без горизонтального padding). Таким образом пользовательские блоки и AI-контент будут выровнены одинаково — таблицы обоих начинаются от края 600px контейнера.

Также убрать `p-4` у contentEditable (строка 253), заменить на пустую строку — чтобы AI-контент тоже не имел лишнего padding. Таблицы внутри обоих типов блоков уже содержат собственные отступы.

```
// Строка 253: было "p-4 outline-none ..." → станет "outline-none ..."
// Строка 391: было "px-4 py-1" → станет "py-1"
```
