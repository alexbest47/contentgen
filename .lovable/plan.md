

## Проблема

Страница `/programs/:programId/offers/diagnostic` (`OfferTypeDetail`) показывает записи из таблицы `offers`, а страница `/diagnostics` — из таблицы `diagnostics`. При удалении диагностики удаляется только запись из `diagnostics`, но не из `offers`, поэтому данные расходятся.

## Решение

Для типа `diagnostic` страница `OfferTypeDetail` должна показывать данные из таблицы `diagnostics`, а не из `offers`. Это проще и надёжнее, чем синхронизировать две таблицы.

### Изменения в `src/pages/OfferTypeDetail.tsx`

Когда `offerType === "diagnostic"`:
- Запрашивать `diagnostics` с фильтром `program_id` вместо `offers`
- Показывать статус диагностики (бейдж)
- Навигировать по клику на `/diagnostics/:id` (страница деталей диагностики)
- Кнопку «Создать» вести на `/create-diagnostic` (с предвыбранной программой)
- Скрыть кнопку редактирования оффера (редактирование через страницу диагностики)
- Удаление — удалять из `diagnostics` (и связанный `offer` через `offer_id`)

Для всех остальных типов офферов — поведение не меняется.

### Изменения в `src/pages/Diagnostics.tsx`

Никаких — админская страница продолжает работать как есть.

### Изменения в `src/pages/CreateDiagnostic.tsx`

Поддержать query-параметр `?programId=...` для предвыбора программы при переходе со страницы программы.
