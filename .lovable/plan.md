

## Замена хардкодного маппинга на поиск по имени в БД

### Проблема
Функция `import-prompts-txt` использует жёстко заданный словарь `mapSection()` для преобразования заголовков секций в `content_type` + `channel`. Если название в файле хоть немного отличается — секция не распознаётся. Это хрупкое решение, которое ломается при каждом новом типе контента или переименовании промпта.

### Решение
Убрать `mapSection()` полностью. Вместо этого — искать промпт напрямую по `name` в таблице `prompts`:

```
-- Для каждой секции:
SELECT id, name FROM prompts WHERE name = '<section_name>' LIMIT 1
```

Если промпт найден по имени — обновляем его `system_prompt` и `user_prompt_template`. Если не найден — добавляем в `notFound`.

### Изменения

**`supabase/functions/import-prompts-txt/index.ts`**:
- Удалить функцию `mapSection()` и весь связанный код (определение маппинга, условная логика по `content_type`/`channel`)
- В цикле по секциям: заменить маппинг на `supabase.from("prompts").select("id, name").eq("name", section.name).limit(1)`
- Если промпт найден — обновить; если нет — добавить в `notFound`
- Убрать массив `unmapped` (больше не нужен — всё либо найдено, либо нет)

Это делает импорт полностью совместимым с экспортом: экспорт пишет `p.name`, импорт ищет по тому же `name`.
