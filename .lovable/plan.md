

## Пунктирная рамка вокруг всего баннера для background-image плейсхолдеров

### Что изменится

Когда изображение не выбрано — вместо серой заглушки в левой ячейке, весь баннер (обе ячейки) обрамляется пунктирной рамкой с текстом «header_banner — 600×200» и кнопками «Сгенерировать» / «Из библиотеки» по центру. Когда изображение выбрано — рамка исчезает, обе ячейки показывают `background-image`.

### Технические детали

**Файл: `src/components/email-builder/BlockCanvas.tsx`**

**1. `preprocessHtmlWithPlaceholders`** — изменить обработку unfilled background-image плейсхолдеров:
- **Первое вхождение (unfilled)**: вместо замены на `background-image: none; background-color: #e5e7eb` внутри ячейки — найти ближайшую родительскую таблицу/`<table>` или строку, и обернуть весь блок в контейнер с `data-placeholder-id` и `data-placeholder-unfilled`. Убрать `background-image` из стиля ячейки но НЕ добавлять серый фон — оставить ячейку прозрачной.
- **Последующие вхождения (unfilled)**: просто убрать `background-image: none`, без дополнительных атрибутов (как сейчас).
- **Filled (оба вхождения)**: без изменений — подставлять реальный URL.

Упрощённый подход: вместо сложного поиска родительской таблицы, добавить data-атрибут `data-placeholder-id` и `data-placeholder-unfilled` на ПЕРВЫЙ элемент как сейчас. Затем в `measurePlaceholders` — когда находим unfilled элемент с background-image, подниматься по DOM к ближайшему `<table>` предку и использовать его rect для позиционирования оверлея.

**2. `measurePlaceholders`** — для unfilled элементов с `data-placeholder-unfilled`:
- Проверять, есть ли у элемента `<table>` предок внутри контейнера. Если есть — использовать rect таблицы (баннер целиком) вместо rect одной ячейки.
- Это автоматически позиционирует кнопки по центру всего баннера.

**3. Оверлей для unfilled background-image плейсхолдеров** — в JSX рендеринге unfilled оверлеев:
- Добавить пунктирную рамку (`border: 2px dashed #ccc; border-radius: 8px`) на весь оверлей.
- Показать текст с типом/размером и кнопки по центру (как уже делается для div-плейсхолдеров).

Различие от div-плейсхолдеров: для background-image unfilled нужна отдельная стилизация оверлея (пунктир + полупрозрачный фон), чтобы содержимое баннера (тёмная правая ячейка с текстом) было видно под ним.

**4. `restorePlaceholderMarkers`** — без изменений (уже корректно работает).
