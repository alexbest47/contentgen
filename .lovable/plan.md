

## Исправить описание и данные переменных офферов

### Проблема
Сейчас на странице «Переменные промптов» карточки для промокодов, «Освободилось место», «Старт нового потока» и «Вебинары» показывают JSON-массив **всех** активных офферов данного типа. Но по логике системы (аналогично `{{template_name}}`) эти переменные должны содержать данные **одного конкретного выбранного** оффера, а не массив всех.

### Решение

**`src/pages/PromptVariables.tsx`**

1. Обновить описания в каталоге `categories`:
   - `{{promo_codes_data}}` → «JSON-данные выбранного промокода (программа, описание, код, дата истечения)»
   - `{{spot_available_data}}` → «JSON-данные выбранного оффера «Освободилось место» (программа, название)»
   - `{{new_stream_data}}` → «JSON-данные выбранного оффера «Старт нового потока» (программа, название, дата старта)»
   - `{{webinar_data}}` → «JSON-данные выбранного вебинара (программа, название, дата, автовебинар, ссылка на лендинг)»

2. Обновить карточки (`PromoCodesCard`, `SpotAvailableCard`, `NewStreamCard`, `WebinarDataCard`):
   - Вместо показа JSON-массива всех офферов — показывать **пример** JSON-объекта одного оффера
   - `CardDescription` изменить с «JSON-массив всех активных...» на «Данные конкретного оффера, выбранного пользователем при генерации»
   - Если есть хотя бы один оффер — показать первый как пример формата; если нет — заглушка

3. Обновить `source` в каталоге на «Выбирается пользователем при генерации» вместо «offers (offer_type = ..., is_archived = false)»

### Итого
- 1 файл (`PromptVariables.tsx`), ~30 строк изменено (описания + логика отображения)
