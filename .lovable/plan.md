

## Исправление путаницы в статусах для справочных материалов

### Проблема
Статусы проекта (`generating_leads`, `leads_ready`, `lead_selected`) всегда отображаются как «Генерация лид-магнитов», «Лид-магниты готовы», «Лид-магнит выбран» — даже если `content_type === "reference_material"`. На скриншоте видно: проект помечен как «Справочный материал», но статус — «Лид-магнит выбран».

### Решение

**`src/pages/OfferDetail.tsx`** — сделать `statusLabels` зависимыми от `content_type` проекта:

Вместо статического маппинга, при рендере строки проекта подставлять нужный текст:

| Статус | lead_magnet | reference_material |
|---|---|---|
| `generating_leads` | Генерация лид-магнитов | Генерация справочного материала |
| `leads_ready` | Лид-магниты готовы | Справочный материал готов |
| `lead_selected` | Лид-магнит выбран | Справочный материал выбран |

Остальные статусы (`draft`, `generating_content`, `completed`, `error`) остаются без изменений.

Реализация: добавить функцию `getStatusLabel(status, contentType)` которая возвращает нужный текст. Изменение только в одном файле.
