

## Разделение на два этапа: создание и генерация

### Суть изменения

Сейчас нажатие «Создать» одновременно сохраняет диагностику и запускает генерацию. Нужно разделить:

1. **Этап 1 — Создание**: Форма сохраняет запись в `diagnostics` со статусом `draft`. После сохранения — редирект на страницу диагностики.
2. **Этап 2 — Генерация**: На странице диагностики есть кнопка «Сгенерировать», которая запускает весь пайплайн (Claude → изображения → финальный JSON).

### Что меняется

**`src/pages/CreateDiagnostic.tsx`** — упрощается до чистой формы:
- Убрать всю логику генерации (`handleGenerate`, `steps`, `viewState`, `resultJson`, прогресс-UI, error/result views)
- Кнопка «Создать» вставляет запись в `diagnostics` (статус `draft`) + оффер в `offers`, затем `navigate` на новую страницу диагностики

**Новая страница `src/pages/DiagnosticDetail.tsx`**:
- Загружает диагностику по `id` из URL
- Показывает информацию (название, описание, программа, теги, промпт)
- Кнопка «Сгенерировать» — запускает пайплайн (вся логика генерации переносится сюда: вызов `generate-diagnostic`, затем `generate-diagnostic-images`, прогресс-UI, результат с JSON)
- После генерации показывает результат (JSON, копировать, скачать)
- Если уже сгенерировано (`status = ready`) — сразу показывает результат

**`src/App.tsx`** — добавить маршрут `/diagnostics/:diagnosticId` (adminOnly)

**Миграция** — добавить значение `draft` как дефолтный статус (уже `generating`, нужно просто использовать `draft` при вставке)

### Порядок реализации

1. Упростить `CreateDiagnostic.tsx` — только форма + сохранение + редирект
2. Создать `DiagnosticDetail.tsx` — просмотр + кнопка генерации + прогресс + результат
3. Добавить маршрут в `App.tsx`
