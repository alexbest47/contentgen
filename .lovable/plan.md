

## Выбор проекта в правой панели конструктора писем

### Проблема

Сейчас код автоматически берёт **последний** проект для комбинации `offer_id + content_type` (`limit(1)`). Но для одного оффера может быть много проектов с разными вариантами (как на скриншоте — 10+ проектов для «Предварительный список»). Пользователь не может выбрать нужный проект или создать новый.

### Решение

Добавить в правую панель **dropdown выбора проекта** между полем «Оффер» и блоком выбора варианта.

### Изменения в `GeneratedBlockSettings.tsx`

**1. Запрос всех проектов вместо одного:**

Заменить `variantProject` query (строки 84-97): вместо `limit(1)` загрузить **все** проекты для `offer_id + content_type`, включая `title` и `created_at`.

Аналогично для `angleProject` (строки 135-154) — загружать все проекты с нужными фильтрами.

**2. Новое состояние `selected_project_id` в config:**

Хранить выбранный проект в `config.selected_project_id`. Сбрасывать при смене оффера.

**3. Новый UI-блок — dropdown проектов:**

После выбора оффера показывать:
- Dropdown со списком проектов (название + дата). 
- Последний пункт: «+ Создать новый проект» — при выборе создаёт проект и вызывает `generate-lead-magnets`.

Если проектов нет вообще — показывать только кнопку «Сгенерировать варианты» (текущий Сценарий А).

**4. Загрузка вариантов из выбранного проекта:**

Query `variants` использует `config.selected_project_id` вместо автоматического `variantProject?.id`.

**5. Сброс при смене оффера:**

При смене `offer_id` сбрасывать `selected_project_id` и `selected_variant_id`.

### Для двухуровневых блоков

Та же логика: после выбора кейса/возражения — dropdown проектов с углами, с возможностью создать новый.

### Итого

Один файл: `src/components/email-builder/GeneratedBlockSettings.tsx`. Точечные правки в queries и добавление одного Select-блока.
