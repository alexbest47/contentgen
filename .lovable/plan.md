

## План: Google Docs ссылка вместо текстового поля описания аудитории

### Суть изменения
Заменить текстовое поле "Описание аудитории" на поле ввода URL Google документа. Система будет автоматически загружать содержимое документа при генерации контента.

### Изменения в БД
- Добавить колонку `audience_doc_url TEXT` в таблицу `mini_courses`
- Колонка `audience_description` останется — в неё будет сохраняться извлечённый текст из документа (кэш)

### Изменения в UI (`ProgramDetail.tsx`)
- Заменить `Textarea` для "Описание аудитории" на `Input` с placeholder "https://docs.google.com/document/d/..."
- Добавить подсказку: "Вставьте ссылку на Google документ (доступ по ссылке для всех)"
- При создании мини-курса сохранять URL в `audience_doc_url`

### Новая Edge-функция `fetch-google-doc`
- Принимает URL Google документа
- Извлекает document ID из URL
- Загружает содержимое через Google Docs export API: `https://docs.google.com/document/d/{ID}/export?format=txt`
- Возвращает текст документа

### Изменения в `generate-lead-magnets`
- Перед генерацией: если есть `audience_doc_url`, вызвать fetch для получения текста
- Сохранить полученный текст в `audience_description` (кэш для повторных запусков)
- Использовать этот текст в промпте как `{{audience_description}}`

### Поток данных
1. Пользователь вставляет ссылку → сохраняется в `audience_doc_url`
2. При генерации лид-магнитов → edge-функция загружает документ по URL → подставляет текст в промпт
3. Текст кэшируется в `audience_description` для просмотра
