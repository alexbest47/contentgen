

## Баг: задача застревает в статусе «Обработка»

**Причина:** В `classify-case` есть 3 ветки выхода, но проверка завершения задачи (`job completion check`) выполняется только в 2 из них:
1. ✅ Успешная классификация — проверяет и ставит `completed`
2. ✅ Дедупликация (skip) — проверяет и ставит `completed`  
3. ❌ **Нет транскрипта** (строки 39-75) — НЕ проверяет завершение задачи
4. ❌ **Catch-блок ошибки** (строки ~230+) — тоже НЕ проверяет

Если последний обработанный файл попадает в ветку «нет транскрипта» или ошибку — задача навсегда остаётся в `processing`.

### План исправления

**`supabase/functions/classify-case/index.ts`** — добавить проверку завершения задачи в два места:

1. **После ветки «No transcript»** (после строки 67): добавить блок проверки `remainingCount` и обновления `case_jobs.status = "completed"` — идентичный тому, что уже есть в success-ветке.

2. **В catch-блоке ошибки** (после цепочки к следующему файлу): добавить аналогичную проверку завершения.

Также: выполнить одноразовое исправление текущей застрявшей задачи через SQL `UPDATE case_jobs SET status = 'completed' WHERE id = '0b6b678a-...'`.
