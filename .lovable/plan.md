

## План: Перенос описания аудитории на уровень платной программы

### Суть
Поле `audience_doc_url` (ссылка на Google Doc с описанием аудитории) переносится с уровня мини-курса на уровень платной программы. Все мини-курсы программы используют одну и ту же аудиторию.

### 1. База данных (миграция)
- Добавить `audience_doc_url TEXT` и `audience_description TEXT` в таблицу `paid_programs`
- Удалить `audience_doc_url` и `audience_description` из `mini_courses` (или оставить для обратной совместимости, но перестать использовать)

### 2. UI: Страница создания программы (`Programs.tsx`)
- Добавить поле "Ссылка на описание аудитории (Google Docs)" в диалог создания программы
- На карточках программ показывать индикатор "📄 Google Doc привязан"

### 3. UI: Страница программы (`ProgramDetail.tsx`)
- Показывать ссылку на Google Doc аудитории в шапке страницы (с возможностью редактирования)
- Убрать поле `audience_doc_url` из формы создания мини-курса — остаётся только название и описание курса

### 4. Edge-функция `generate-lead-magnets`
- При получении данных для генерации брать `audience_doc_url` и `audience_description` из `paid_programs` (через связь `mini_courses.program_id → paid_programs`)
- Логика загрузки Google Doc остаётся без изменений, только источник URL меняется

### Затрагиваемые файлы
- Новая миграция SQL
- `src/pages/Programs.tsx` — добавить поле URL в форму создания
- `src/pages/ProgramDetail.tsx` — убрать поле аудитории из формы мини-курса, показать URL аудитории в шапке
- `supabase/functions/generate-lead-magnets/index.ts` — читать аудиторию из `paid_programs`
