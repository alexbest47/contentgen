

## Задача

Добавить фильтрацию и поиск в селекторе кейсов на этапе пре-генерации письма. 206 кейсов — обычный Select неудобен.

## Данные

- `mentioned_products` — пустые у всех кейсов, но есть `video_type` (отзыв, кейс, промо, выпускное, практика, образовательный)
- `tags` — массив строк, богатый набор (декрет, психоанализ, нутрициолог и т.д.)
- `student_name` — есть не у всех

## Решение

Заменить простой `<Select>` в `LetterGenerationPanel.tsx` на компонент с:

1. **Текстовый поиск** — `<Input>` сверху, фильтрует по имени файла, имени студента и тегам (подстрока)
2. **Фильтр по типу видео** — горизонтальные Badge-кнопки (отзыв, кейс, промо и т.д.), множественный выбор
3. **Список кейсов** — скроллируемый список карточек с именем студента, названием файла, типом и тегами; клик выбирает кейс

### Изменения

**`src/components/email-builder/LetterGenerationPanel.tsx`** (секция pre-generation, строки 146-165):
- Заменить `<Select>` на inline-компонент с:
  - `<Input placeholder="Поиск по имени, файлу или тегам..." />`
  - Ряд Badge для фильтра по `video_type`
  - `<ScrollArea>` со списком кейсов, каждый показывает: имя студента, имя файла, тип, теги (первые 3)
  - Выбранный кейс подсвечивается
- Фильтрация — локальная (все кейсы загружены), по `useMemo`

Один файл, ~80 строк изменений.
